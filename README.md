

# 地址

区块链上的地址通常源自助记词（mnenomic phrases）。助记词通常是随机生成的单词列表，这些单词及其特定的顺序通过算法计算出私钥，然后可以使用该私钥来派生公钥，从而为我们提供密钥对（也称为私钥和公钥的组合）。

> 助记词通常是根据 BIP39 标准生成的，共计包含 2048 个单词，见 [BIP39 单词列表](https://github.com/bitcoin/bips/blob/master/bip-0039/bip-0039-wordlists.md)

从助记词派生的私钥（Private Key）是特殊密钥，是唯一能够为其相应公钥签署交易的密钥，从而证明它拥有公共地址。

> 私钥无法反推出助记词！

单个助记词可以通过不同的派生路径（derivation path）派生出多个公钥，而这些公钥无法辨别它们来自同一个私钥。

> 公钥也无法反推出私钥！

派生路径本质上是一个遵守 BIP32 标准，格式为 `m/<PURPOSE>/<COIN_TYPE>/<ACCOUNT>/<CHANGE>/<ADDRESS_TYPE>` 的序列：
- 每个元素通过 `/` 分隔。`m` 表示根密钥或“主”密钥
- `m` 之后的每个其他元素都应该是零或正整数
- `<PURPOSE>` 定义钱包的用途，常见的值：
  - `44'`：用于传统比特币地址（Legacy）
  - `49'`：用于嵌套的SegWit地址（Nested SegWit）
  - `84'`：用于原生SegWit地址（Native SegWit）
- `<COIN_TYPE>` 区分不同的区块链，如比特币是 0，莱特币是 2
- `<ACCOUNT>` 区分不同账户
- `<CHANGE>` 0 代表外部地址，1 代表内部地址
- `<ADDRESS_TYPE>` 表示具体的地址索引号，用于生成一系列的地址

如比特币的 `m/44'/0'/0'/0/0` 表示按照 BIP44 标准、比特币币种、账户 0、外部地址，生成第一个地址的公私钥对。
